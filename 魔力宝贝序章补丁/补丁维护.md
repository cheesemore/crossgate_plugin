# 补丁维护

**本目录** `魔力宝贝序章补丁/` 是序章热补丁工具的唯一维护位置。  
当前 hotfix 基准体积：**7,068,160** 字节（`cg37_Data/assets/hotfixdata/hotfix.dll.bytes`，2026-07-19）。

---

## 1. 怎么用（玩家）

1. 完全关闭 `cg37.exe`
2. 双击 **`启动补丁GUI.bat`**
3. 选择游戏目录（含 `cg37_Data`）
4. 点红色 **「初始化」**（编译引擎、同步底稿、恢复干净 hotfix / `.orig`）
5. 勾选补丁 → **「应用补丁」** → 启动游戏验证

可选：**「测算余量」** 预检 `.text` VA 间隙是否够打依赖追加 IL 的补丁。

### 互斥规则

- **IL 九动** ⊥ **外挂九动** ⊥ **助手桥接**  
  （共用 `HotfixEntry.OnApplicationPause` / `.text` 余量）

### 外挂九动（重点）

| 项 | 说明 |
|----|------|
| 源码 | `tools/seqchapter_nine_action/SeqChapterNineAction.cs` |
| 部署文件 | `{hotfixdata}/SeqChapterNineAction.dll.bytes` |
| 加载 | 与桥接同类：`OnApplicationPause` 加载器 → `Bootstrap` |
| 扩队列 | **`OnCommandPlayerCallback` 末尾同步**调用 `OnCommandPlayerEnd` → `ExpandAccountList`（**不用 Timer**） |
| Magics | 仍改 hotfix 内 `PlayerActionMagics` 赋值（`ldc.i4.1→0`） |
| 挂载状态 | `%USERPROFILE%\.seqchapter_helper\nine_action.status`（如 `mounted=sync_hook_ok` / `expanded=...` / `expand_error=...`） |
| 余量测算 | `HotfixPatcher slack-report --hotfix <.orig>`；GUI「测算余量」 |

**为何有外挂版**：新版 hotfix `.text`→`.rsrc` 虚拟间隙过小，IL 整法追加会侵节或后迁方法导致启动未响应；外挂把队列逻辑放到 DLL，hotfix 只留加载钩 + Magics。

**弊端（相对 IL 原版）**：反射链路更长、多文件、与桥接互斥、Cecil 写出后需再叠原地补丁，排错要查状态文件。间隙够时应优先 IL。

组合应用顺序（引擎侧）：**外挂九动先打**（Cecil），再 VIP / Sprint 等原地补丁。

## 4. 核心约束

1. **hotfix 文件字节数不变**（`HotfixSize.Expected` / `EXPECTED_SIZE`）。
2. **可执行追加 IL**受两道墙限制：
   - **VA 间隙** = `.text` 下一节 VirtualAddress −（`.text` VA + VirtualSize）
   - **raw slack** = `.text` SizeOfRawData − VirtualSize（文件尾零填充）  
   测算里的 `usable_append` ≈ `min(va_gap, raw_slack)`。  
   实际写入时，`BinaryPeWriter` **可从 `.reloc`/`.rsrc` 压缩腾 raw**（文件总长仍不变），故 **VA 够、raw 不够** 时 IL 九动仍可能打进；预检若只看 `usable` 会偏紧（见 `patch_slack.py` 对 IL 九动的放宽）。
3. **禁止后移** `.rsrc`/`.reloc` 的 VirtualAddress（与 DataDirectory 脱节 → 启动未响应）。
4. **禁止后迁带 EH 的邻居方法**进 VA 间隙（实测闪退/未响应）。
5. 多数玩法补丁应优先 **原地改写**；新增 IL 优先塞进 VA 间隙。

### 4.1 余量会不会变成 0？

**会。** 某次客户端更新后完全可能出现：

| 情况 | 含义 | 对基础功能的影响 |
|------|------|------------------|
| `va_gap = 0`（或极小） | `.text` 虚拟末尾贴死下一节 | **IL 追加类**（特效加速、IL 九动、桥接加载器）打不进或极危险 |
| `raw_slack = 0` 且无法再压缩邻节 | 文件内腾不出可执行字节 | 同上；即便 VA 有空也写不进 |
| 两者都 &gt; 0 但 &lt; 需求 | 例如特效约 49B、IL 九动约 603B | 部分能打、部分不能 |

**基础功能里相对「抗更新」的**（不依赖余量）：

- VIP / 非VIP 战斗倍速、Sprint、战斗长按、客服改开（缩体/换 call）→ **原地改写**，余量为 0 仍通常可打。

**依赖余量的**：

- 技能特效加速（约 +49B）
- 神奇九动 IL（约 +603B）或外挂九动加载钩（约 +180B 量级）
- 助手桥接加载器（约 +200B）

因此：**余量为 0 不会让整套工具废掉，但会逼停「追加 IL」路线**；九动应切外挂 DLL（钩子更短时仍可能要一点点余量），特效加速可能要暂时关掉或改成别的手法。

历史对照：2026-07-15 前后 `va_gap≈438`、IL 九动打不进；2026-07-18 `va_gap≈4228`，IL 九动又可用。余量**可升可降**，每次更新都要重新 `slack-report`。

## 5. 基础功能保底（更新后必保）

### 5.0 什么叫「基础功能」

以下组合为目标保底（与近期实装一致）：

| 项 | 默认 | 余量依赖 |
|----|------|----------|
| VIP / 非VIP 战斗倍速 **5x** | 开 | 无（原地） |
| Sprint **8** | 开 | 无 |
| 战斗长按详情 | 开 | 无 |
| 技能特效 **2x** | 开 | 约 +49B |
| 客服 → **自动技能** | 开 | 无（通常缩体） |
| 神奇九动（**IL 或外挂二选一**） | 需勾选 | IL ≈603B VA；外挂钩子更小 |

互斥：九动 ⊥ 助手桥接。保底组合**默认不绑桥接**。

### 5.1 用户侧（启动器「更新 / 修复」后）

```
1. 关闭所有 cg37.exe
2. 启动器更新或修复到最新
3. 打开补丁 GUI：若提示「检查到游戏有更新，是否自动修复？」→ 选「是」
   （采用游戏内干净新 hotfix 重建底稿；不会用旧 .orig 盖新文件）
4. 「应用补丁」→ 冒烟验证
```

指纹文件：`魔力宝贝序章补丁/hotfix_watch_stamp.json`（每次初始化 / 打补丁成功后更新）。  
若提示体积不对 / 自动修复失败 → 交给开发者按 §5.2 改常量。

### 5.2 开发者 / Agent：每次更新标准流程

**目标：尽快恢复 §5.0，而不是一次做完所有可选功能。**

```
A. 量体积
   干净 hotfix.dll.bytes 字节数 → 与 HotfixSize.Expected / EXPECTED_SIZE 对比

B. 体积变了 → 改常量并重建
   - tools/hotfix_patcher/HotfixSize.cs → Expected
   - 魔力宝贝序章补丁/scripts/patch_common.py → EXPECTED_SIZE
     （KNOWN_OLD_SIZES 追加上一版）
   - 序章助手共享/scripts/check_client_integrity.py 体积提示（若有）
   - 本文件文末历史表
   - 重建补丁引擎.bat
   - 复制干净 hotfix → tools/hotfix.dll.bytes.neworig
   - 更新 tools/hotfix_baseline.json

C. 嗅探目标是否还在（对 .orig / neworig）
   HotfixPatcher customer-gm-patch --hotfix <.orig> --sniff
   HotfixPatcher slack-report --hotfix <.orig> --json
   关键目标（丢一个修一个）：
   - BattleManager.get_BattleTimeScale（VIP）
   - CharacterEntity.set_Running（Sprint）
   - BattleRole.OnLongPress（长按）
   - EffectEntity.Play（特效）
   - MapSidebarPanel.OnClickCustom + AutoSkillSettingPanel / BattleAutoSkillManager
   - BattleProcesser.OnCommandPlayerCallback + PlayerActionMagics 三处（九动）

D. 副本自测（禁止直接当最终交付写游戏目录，除非用户明确要求代打）
   1) 只打原地组：VIP+Sprint+长按+客服自动技能 → 须成功、体积不变
   2) 再加特效 2x → 若失败，记录余量，保底可暂时关特效
   3) IL 九动；失败则改外挂九动（工作目录须在 hotfixdata，以便编 DLL）
   4) 「原地组 + 特效 + 一种九动」整组 → 体积不变

E. 交付用户
   关闭游戏 → GUI 初始化 → 应用基础组合 → 进游戏冒烟
```

### 5.3 体积不变但 hash 变了

- GUI「初始化」同步底稿即可。  
- 某补丁失败：ildump 对照目标方法，改对应 `*IlPatcher.cs`，重建引擎。  

### 5.4 外挂九动专项（更新后检查清单）

| 检查项 | 做法 |
|--------|------|
| 编译 DLL 引用 | hotfixdata 内仍有 `mscorlib.dll.bytes` / `system*.dll.bytes` |
| 加载钩仍可用 | `ildump HotfixEntry.OnApplicationPause` 原版近 `ret`；Start/Quit/GM.Start 挂点仍在 |
| 同步 Expand 点 | `OnCommandPlayerCallback` 仍填 `AcountList`；末尾仍有可插入的 `ret` |
| 自测挂载 | 打补丁后看 `nine_action.status` 是否 `mounted=sync_hook_ok` |
| 自测扩写 | 进战（非观战/回放）后 status 出现 `expanded=n=…`；失败看 `expand_error` |
| IL 版是否恢复可行 | `slack-report`：`va_gap` ≥ 约 603B 时可优先 IL 九动 |

### 5.5 常见故障

| 现象 | 处理 |
|------|------|
| 体积应为 xxx | 更新 `EXPECTED_SIZE` / `HotfixSize` 并重建引擎 |
| 启动未响应 / 黑屏 | 恢复 `.orig`；查是否越 VA 间隙或后移节区 VA |
| 注入后闪退 | 恢复原版 hotfix → 原版能进再逐项重打；查 `nine_action.status` / 桥接 log |
| 外挂九动进战闪退 | 确认用最新引擎；先只打外挂九动；再看 `expand_error` |
| 助手连不上 | 未勾桥接，或勾了九动互斥掉了桥接 |
| 余量测算拦 IL 九动但 va_gap 很大 | 预检已按 VA 放宽；仍失败再查 raw 压缩 / 改外挂九动 |
| 余量≈0 特效/九动都失败 | 先保「原地组 + 客服」；九动改外挂；特效可关 |

## 6. 开发与发布

```bat
:: 重建引擎
魔力宝贝序章补丁\重建补丁引擎.bat

:: 余量
HotfixPatcher slack-report --hotfix <路径\hotfix.dll.bytes.orig>

:: 反汇编
HotfixPatcher ildump <hotfix> BattleProcesser.OnCommandPlayerCallback

:: 外挂九动单独打（对副本或由 GUI 应用）
HotfixPatcher battle-nine-external-patch --hotfix <.orig> --output <hotfix>

:: 发布 zip
魔力宝贝序章补丁\发布\发布.bat
```

新增普通补丁：在 `tools/hotfix_patcher/` 加 `*IlPatcher.cs` → `Program.cs` 注册 → 同步 `scripts/`（`patch_common` / `apply_combo_patch` / GUI）→ 重建 → 仅对副本 / `.orig` 自测。

新增或改外挂九动：改 `tools/seqchapter_nine_action/` 与 `BattleNineActionExternalIlPatcher.cs` → 重建 → 探针启动 + 进战。
