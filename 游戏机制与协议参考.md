## 1. 生命储备 / 魔力储备

### 1.1 各角色当前储备量（数值）

| 字段 | 类型 | 说明 |
|------|------|------|
| `PlayerData.hpPond` | `int` | 生命储备（血池剩余） |
| `PlayerData.mpPond` | `int` | 魔力储备（魔池剩余） |

**读取方式：**

```csharp
// 单个角色
PlayerData p = PlayerDataHolder.GetPlayerFromUid(uid);
int hpReserve = p.hpPond;
int mpReserve = p.mpPond;

// 全部已加载角色
foreach (var kv in PlayerDataHolder.GetAllPlayers())
{
    string uid = kv.Key;
    int hp = kv.Value.hpPond;
    int mp = kv.Value.mpPond;
}
```

**同步来源：** 服务端推送 `Proto_SC_UpdatePlayer` → `RoleManager` 按 `PROTO_PLAYER_TYPE` 更新：

- `PROTO_PLAYER_TYPE_HPPOND` → `HpPond` → `hpPond`
- `PROTO_PLAYER_TYPE_MPPOND` → `MpPond` → `mpPond`

### 1.2 血魔池上限（账号级）

| 字段 | 说明 |
|------|------|
| `PlayerDataHolder.HpMpPoolMax` | 血魔池容量上限 |

**协议：** `LSSPROTO_HPFP_FUNC = 1019`

**主动查询：**

```csharp
Manager<RoleManager>.Instance.OpenHpMpPool(uid);
// 等价于 SendHpFp("血魔池状态", uid)
```

**服务端回包：** `Proto_SC_HpFp`，`Type == "血魔池状态"` 时写入 `PlayerDataHolder.HpMpPoolMax = proto.Max`。

### 1.3 各角色开关状态（血池 / 魔池 开/关）

开关状态有两套数据结构，用途不同：

#### A. Buff 面板（主界面血池/魔池图标，buff id 5 / 6）

存储在 `RoleManager.m_buffInfo`（`Dictionary<string, Proto_SC_PlayerBuff>`）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `HpStatus` | `bool` | 血池开关（true=开启） |
| `FpStatus` | `bool` | 魔池开关（true=开启） |
| `HpfpHp` / `HpfpFp` | `int` | 池内当前 HP/MP |
| `HpfpMaxhp` / `HpfpMaxfp` | `int` | 池内上限（进度条用） |

**协议：** `LSSPROTO_PLAYERBUFF_FUNC = 1020`（服务端推送为主）

**读取：**

```csharp
if (Manager<RoleManager>.Instance.m_buffInfo.TryGetValue(uid, out var buff))
{
    bool hpPoolOn = buff.HpStatus;
    bool mpPoolOn = buff.FpStatus;
}
```

**UI 逻辑：** `PlayerBuffPanel` 中 buff `Id == 5` 为血池、`Id == 6` 为魔池；点击开关发送：

- `"关闭血池"` / `"开启血池"`
- `"关闭魔池"` / `"开启魔池"`

**主动拉取（仅见 buff 倒计时到期时）：**

```csharp
Proto_CS_PlayerBuff req = new Proto_CS_PlayerBuff();
req.Type = "玩家BUFF数据";
// 可选 req.KUid = uid;
Manager<NetManager>.Instance.SendMessage(LSSPROTO.LSSPROTO_PLAYERBUFF_FUNC, req);
```

> 多角色 buff 数据通常由服务端在登录/切号时推送；客户端平时只在 buff 计时结束时主动请求一次（未带 KUid）。

#### B. 系统设置 → 战斗页（阈值 + 总开关）

**查询：** `SendHpFp("血魔池设置", uid)` → 回包 `Proto_SC_HpFp`，`Type == "血魔池设置"`

| 字段 | 类型 | 说明 |
|------|------|------|
| `LowRateHp` / `LowRateFp` | `int` | 自动补充阈值（百分比） |
| `HpStatus` / `FpStatus` | `int` | 设置页中的开关状态 |
| `Hppool` / `Fppool` | `int` | 池数值 |
| `KUid` | `string` | 角色 uid |

**设置命令（`RoleManager.SendHpFp`）：**

| Type 字符串 | 含义 |
|-------------|------|
| `"血魔池状态"` | 查容量上限 |
| `"血魔池设置"` | 查/刷新设置页数据 |
| `"血魔池记录"` | 查使用记录 |
| `"开启血魔池"` / `"关闭血魔池"` | 设置页总开关 |
| `"开启血池"` / `"关闭血池"` | 单独血池开关 |
| `"开启魔池"` / `"关闭魔池"` | 单独魔池开关 |
| `"血池阈值"` / `"魔池阈值"` | 修改阈值（带 rate 参数） |
| `"获取血池道具"` / `"获取魔池道具"` | 打开补充道具界面 |

**入口代码：**

- `RoleManager.SendHpFp(...)` → `LSSPROTO_HPFP_FUNC`
- `RoleManager.OnHpFpCallback` 分发 `ROLE_TYPE_EVENT.HPFP`
- `SystemSettingPanel.CheckHpMpState(uid)` 在切角色时调用

## 2. 战斗协议与游戏状态

### 2.1 进入战斗（服务端 → 客户端）

| 项目 | 值 |
|------|-----|
| 协议号 | `LSSPROTO_CREATE_BATTLE_FUNC = 4` |
| 消息体 | `Proto_SC_CreateBattle` |
| 处理 | `BattleProcesser.OnCreateBattleCallback` → 协程 `EnterBattle` |

**关键流程：**

1. `info.Result > 0` 时视为有效战斗
2. `BattleDataHolder.battleModeFlag = (BATTLE_TYPE)info.Result`
3. `EncountFlag = true`（遇敌过渡）
4. `EnterBattle` 协程末尾：`BattleDataHolder.IsInBattle = true`
5. 进入前会 `walkSystem.SaveAndStopCurRequest()` 保存并停止寻路

**PVP 另有：** `LSSPROTO_CREATE_BATTLE_PVP_FUNC = 7`

### 2.2 战斗结束

战斗结束分两步，不是同一个包立刻清状态：

| 阶段 | 协议号 | 作用 |
|------|--------|------|
| 战斗逻辑结束 | `LSSPROTO_BATTLE_END_FUNC = 75` | `OnBattleEndCallback` → `BattleRelpayManager.isBattleEnd = 1` |
| 退出战斗场景 | `LSSPROTO_BATTLE_COMMAND_EXIT_FUNC = 72` | `OnCommandExitCallback` → `ForceExitBattle()` → 协程 `ExitBattle` |

**`ExitBattle` 协程末尾：**

- `BattleDataHolder.IsInBattle = false`
- `BattleDataHolder.battleModeFlag = BATTLE_TYPE.NONE`
- 派发 `EventCenter.ExitBattle`

**相关战斗协议（战斗中指令流）：**

| 协议号 | 名称 |
|--------|------|
| 71 | `LSSPROTO_BATTLE_COMMAND_STATUS_FUNC` |
| 72 | `LSSPROTO_BATTLE_COMMAND_EXIT_FUNC` |
| 73 | `LSSPROTO_BATTLE_COMMAND_ACTION_FUNC` |
| 74 | `LSSPROTO_BATTLE_CONFIG_FUNC` |
| 75 | `LSSPROTO_BATTLE_END_FUNC` |

### 2.3 如何判断当前游戏状态

| 标志 | 类型 | 含义 |
|------|------|------|
| `PlayerDataHolder.GameType` | `GAME_TYPE` | `LOGIN` / `GAME`（是否在大世界） |
| `BattleDataHolder.IsInBattle` | `bool` | **是否在战斗中**（最常用） |
| `BattleDataHolder.battleModeFlag` | `BATTLE_TYPE` | 战斗类型（普通/PVP/观战/回放等） |
| `BattleDataHolder.IsBattleExit` | `bool` | 退出战斗过渡中 |
| `BattleProcesser.EncountFlag` | `bool` | 遇敌切入动画中 |
| `BattleRelpayManager.isBattleEnd` | `int` | 0=进行中，1=已收到 END |
| `PlayerDataHolder.playerData.encounterStatus` | — | 非 0 表示自动挂机中，禁止手动移动/任务寻路 |

**助手推荐判断：**

```csharp
bool inGame   = PlayerDataHolder.GameType == GAME_TYPE.GAME;
bool inBattle = BattleDataHolder.IsInBattle;
bool autoHook = PlayerDataHolder.playerData.encounterStatus != 0;
```

## 3. 寻路（本机逻辑）

寻路核心在客户端 `WalkSystem` + `MapManager.FindPath`（A*），**不经过独立寻路协议**；跨图传送才走 `ServerSyncMove` 等服务端同步。

### 3.1 调用链对比

| 入口 | 代码路径 | 特点 |
|------|----------|------|
| **小地图点击** | `WorldMapPanel.OnLargeMapClick` → `playerEntity.MoveTo(location)` | 同图直接 A*；显示自动寻路面板 |
| **世界地图导航** | 设目标 → `gotoTarget()` → `GeneralPointMoveTo` | 先 `CanReachTo` 检测；不可达则走 `GeneralPointPathConfig` 跨图路径，必要时 `TaskManager.StartWayThMissionStepByID` |
| **任务寻路** | `TaskManager.StartWayThMissionStepByID` → `RunTask` → `MissionSystem.TaskMoveTo` | 带任务脚本、`ignoreHalfwayWarp: true`、到达回调驱动任务步骤；可跨图 warp |

**结论：** 底层都是 `WalkSystem.MoveTo`，但**入口不同**——小地图是同图短路径；任务寻路有 Mission 状态机、跨图 PathID、NPC 交互等额外逻辑；世界地图导航介于两者之间（支持跨图 GeneralPoint 配置）。

### 3.2 核心 API

```csharp
// 同图坐标
WalkSystem.MoveTo(Vector2Int target, Action<bool, EWalkError> cb, bool ignoreHalfwayWarp = false)

// 跨图 MapPoint
WalkSystem.MoveTo(MapPoint target, Action<bool, EWalkError> cb, bool ignoreHalfwayWarp = false)

// 任务专用（会先 StopMove，再 MoveTo + ignoreHalfwayWarp）
MissionSystem.TaskMoveTo(MapPoint target, Action<bool, EWalkError> cb)

// 世界地图跨图
CharacterEntity.GeneralPointMoveTo(MapPoint target, Action<bool, EWalkError> cb)
```

### 3.3 枚举值（「方法/状态一共有哪些值」）

#### `EWalkState` — 单次寻路请求状态机

| 值 | 名称 |
|----|------|
| 0 | `Enter` |
| 1 | `BeginGeneralPath` |
| 2 | `OnGeneralPath` |
| 3 | `WaitTransport` |
| 4 | `BeginFindPath` |
| 5 | `OnPathFind` |
| 6 | `Moving` |
| 7 | `Exit` |

#### `EWalkRequestType` — 请求类型

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | `SameMapMove` | 同图移动（小地图点击） |
| 1 | `DifferentMapMove` | 跨图移动 |
| 2 | `ServerSyncMove` | 服务端同步位置 |

#### `EWalkError` — 结果/错误码

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | `None` | 成功 |
| 1 | `CannotFindPath` | 找不到路 |
| 2 | `HaveAtTarget` | 已在目标点 |
| 3 | `TargetCannotReach` | 目标不可达 |
| 4 | `Abort` | 中止 |
| 5 | `CustomCancel` | 自定义取消 |
| 6 | `MapWrong` | 地图错误 |
| 7 | `StateWrong` | 状态错误 |
| 8 | `TransportAbort` | 传送中止 |
| 9 | `SaveAndStop` | 保存并停止（如进战斗前） |

## 4. 商店

### 4.1 协议

| 项目 | 值 |
|------|-----|
| 协议号 | `LSSPROTO_SHOP_FUNC = 1038` |
| 客户端消息 | `Proto_CS_Shop` |
| 服务端消息 | `Proto_SC_Shop` |

### 4.2 典型流程

```
1. SendGetStoreList("商店列表", uid [, tabIndex])
   ← Type="商店列表", Tabs=List<Proto_ShopTabData>

2. SendGetStoreList("商品列表", uid, tabIndex)
   ← Type="商品列表", Items=...

3. SendBuy("购买商品", tabIndex, itemIndex, count, uid, currency)
   ← Type="购买商品"（刷新单件限购等）
```

**代码入口：**

```csharp
Manager<ShopManager>.Instance.Open(uid, shopIndex, itemId);
Manager<ShopManager>.Instance.OpenShop(uid, shopIndex, itemId);
Manager<ShopManager>.Instance.SendGetStoreList(type, uid, tabIndex);
Manager<ShopManager>.Instance.SendBuy("购买商品", tabIndex, itemIndex, count, uid, PROTO_CURRENCY);
```

**`SendBuy` 字段映射：**

- `Proto_CS_Shop.Type = "购买商品"`
- `TabIndex` = 商店页 index
- `Infos.Index` = 商品 index
- `Infos.Num` = 数量
- `Curr` = 货币类型
- `KUid` = 角色 uid

### 4.3 商店切页与「隐藏页」

#### Tab 数据结构 `Proto_ShopTabData`

| 字段 | 说明 |
|------|------|
| `Name` | 页签名称 |
| `Index` | 页签 index（请求商品列表时用） |
| `CurrencyType` | 货币类型 |
| `IsSpecial` | **特殊页标记**（协议字段存在） |

#### UI 行为

- **`StoreMainPanel.RefreshStorePage`**：遍历服务端返回的 `shopType` 列表，**全部显示**，**未**根据 `IsSpecial` 过滤。
- 因此「隐藏页」更可能是：**服务端不下发该 Tab**，或通过 **固定 TabIndex 走独立面板**，而非客户端用 `IsSpecial` 隐藏。

#### 已知特殊 TabIndex（不走普通 StoreMainPanel）

| TabIndex | 面板 / 行为 |
|----------|-------------|
| 5 | `MagicCoreRechargePanel`（魔核充值） |
| 6 | 分享商店（`ShowIntegralUI`，标题「分享商店」） |
| 7 | `FamilyStoreChildPanel` / `PetCollectStoreChildPanel` |
| 8 | 乱斗商店 |
| 9 | 悬赏商店 |

**打开指定页：**

```csharp
Manager<ShopManager>.Instance.OpenShop(uid, shopIndex: 6);  // 直接请求 tab 6 的商店列表
```

**NPC 商店：** `NPCStorePanel` 同样调用 `ShopManager.SendBuy`，UI 有 5 个 Tab Toggle（`m_Tog_Tab_1` ~ `_5`），逻辑与主商店一致。

## 7. 账号每日魔石掉落

> 你观察到的 **非会员 5000 / 会员 20000 / 达上限后不再掉落**，与客户端配置字段和 Buff 展示逻辑一致；**实际上限判定在服务端**，客户端只展示与开关。

### 7.1 配置表（数值来源）

配置在运行时从 AssetBundle 加载，本地调试可读：

`{FileUtil.DataPath}/ExcelGeneral/active_tbfivecharconfig.bytes`  
`{FileUtil.DataPath}/ExcelGeneral/active_tbadvvipcardconfig.bytes`

（需游戏至少下载过配表；若存在 `{DataPath}/我要读取配表` 则走 `ExcelGeneral` 目录。）

#### `FiveCharConfig`（`active_tbfivecharconfig`）

五开账号相关全局配置，**单表一行**：

| 字段 | 含义 |
|------|------|
| `DropLimit` | **账号每日魔石掉落上限（基础值）** → 对应非会员约 **5000** |
| `DropItem` | 掉落物 ID |
| `DropRate` | 掉落率 |
| `DrawItem` / `DefLikeId` | 抽奖/偏好相关 |
| `Title` / `Msg` | 说明文案 |

读取：

```csharp
var cfg = Manager<ConfigManager>.Instance.GetTbFiveCharConfig();
int baseDailyLimit = cfg.DropLimit;
```

#### `AdvVipCardConfig`（`active_tbadvvipcardconfig`）

高级 VIP / 月卡权益配置，**单表一行**：

| 字段 | 含义 |
|------|------|
| `AddDropMoshiLimit` | **会员额外魔石掉落上限** → 与基础相加得 **20000**（5000 + 15000） |
| `Speed` / `RatePetExp` / `RateRoleExp` | 跑速、经验加成 |
| `RemoteSellMoshi` / `FastSellMoshi` | 远程/快速卖魔石 |
| `Message` / `Notice` | 月卡界面说明 |

```csharp
var vipCfg = Manager<ConfigManager>.Instance.GetTbAdvVipCardConfig();
int bonusLimit = vipCfg.AddDropMoshiLimit;
// 会员每日上限 ≈ baseDailyLimit + bonusLimit  （需本地配表确认具体整数）
```

> 客户端 **未** 在 C# 里用 `AddDropMoshiLimit` 做运算或拦截掉落，仅配表存在该字段；上限 enforcement 在服务端。

特权枚举（协议层）：`PROTO_PRIVILEGE_ADD_DROP_MOSHI_LIMIT`。

### 7.2 运行时：当前进度 / 是否还能掉

#### A. Buff 面板（推荐）

服务端通过 **`LSSPROTO_PLAYERBUFF_FUNC = 1020`** 推送 `Proto_SC_PlayerBuff`，缓存于：

```csharp
RoleManager.m_buffInfo[uid]  // Dictionary<string, Proto_SC_PlayerBuff>
```

**魔石相关 Buff：`Proto_BuffInfo.Id == 10`**

| 来源 | 字段 | 说明 |
|------|------|------|
| `Proto_BuffInfo.Value` | int | 服务端下发的数值（可用于当前已掉量，需与 `Time`/文案对照） |
| `Proto_BuffInfo.Time` | int | 常与上限或剩余量一起下发 |
| `Proto_BuffInfo.Str` / `Str2` | string | 标题与描述，**进度条文案多在这里**（如 `3500/5000`） |
| `Proto_SC_PlayerBuff.Moshi` | int | 同步到 `PlayerData.dropMagicStoneSwitch` |

**主动拉取 Buff：**

```csharp
Proto_CS_PlayerBuff req = new Proto_CS_PlayerBuff();
req.Type = "玩家BUFF数据";
req.KUid = uid;  // 建议带上角色 uid
Manager<NetManager>.Instance.SendMessage(LSSPROTO.LSSPROTO_PLAYERBUFF_FUNC, req);
```

查找进度：

```csharp
if (RoleManager.m_buffInfo.TryGetValue(uid, out var buff) && buff.Type == "玩家BUFF数据")
{
    foreach (var info in buff.Info)
    {
        if (info.Id == 10)
        {
            int value = info.Value;   // 当前进度（服务端定义）
            int time  = info.Time;    // 常与上限相关
            string desc = info.Str2;  // UI 描述，常含 "当前/上限"
        }
    }
}
```

#### B. 魔石掉落开关（按角色）

| 存储 | 含义 |
|------|------|
| `PlayerData.dropMagicStoneSwitch` | 来自 `Proto_SC_PlayerBuff.Moshi`；**0 = 开关打开（允许掉）** |
| UI 判断 | `m_StoneSwitch = (dropMagicStoneSwitch == 0)` |

**切换开关（Buff Id 10 UI）：**

```csharp
Manager<RoleManager>.Instance.SendMenu(4, "", uid);
// LSSPROTO_MENU_FUNC = 64, Proto_CS_Menu.Func = 4
```

> 开关只控制「是否参与掉落」，**不**表示是否已达每日上限。

#### C. 会员状态（决定上限是 5000 还是 20000）

| 判断 | 代码 |
|------|------|
| 月卡是否有效 | `PlayerData.MonthCardOpen` → `TimeManager.GetServerTime() < MonthCardTime` |
| 月卡到期时间 | `PlayerData.MonthCardTime` ← `Proto_OnePlayerData.PrivilegeTime` |
| 高级 VIP 图标 | `Proto_PlayerInfo.AdvVipTime1/2 > serverTime` |
| 活动月卡时间 | `Proto_SC_Activity.TqkTime`（月卡子面板 `MonthCardChildPanel`） |

```csharp
bool hasMonthCard = PlayerDataHolder.GetPlayerFromUid(uid).MonthCardOpen;
// 或聊天头像 VIP：AdvVipTime1/2 > TimeManager.GetServerTime()
```

**每日上限估算：**

```csharp
int max = ConfigManager.Instance.GetTbFiveCharConfig().DropLimit;
if (hasMonthCard /* 或 AdvVip 有效 */)
    max += ConfigManager.Instance.GetTbAdvVipCardConfig().AddDropMoshiLimit;
```

（本地无配表时，可暂用你观测值：无会员 **5000**，有会员 **20000**。）

#### D. 未在客户端使用的协议字段

`Proto_ChangeSceneData` 含 `MoshiOpenFlag`、`WorkFlg`（切场景包），**hotfix 内无读取逻辑**，可能仅服务端使用。

### 7.3 排行榜「魔石」≠ 每日进度

排行榜 `Proto_Rank*.Moshi` 用于 **「魔石排行」** 展示（累计类），与 **每日掉落进度** 不是同一字段。

### 7.4 助手实现建议

1. 登录后请求 `"玩家BUFF数据"`，解析 **Buff Id=10** 的 `Value`/`Str2` 得到 **当前/上限**。
2. 用 `MonthCardOpen` 或 `AdvVipTime` 决定上限配表公式。
3. `当前 >= 上限` 时服务端不再给魔石（客户端仅能通过 Buff 文案或停止掉落实测验证）。
4. 需要精确整数时，从 `%LocalAppData%/../` 下游戏 `ExcelGeneral/active_tbfivecharconfig.bytes` 解析，或抓包 `1020` 协议看 Buff 10 字段。

4. IPC/Bridge 直接调用下方 API，无需打开 `CommonlyPanel`。

### 8.2 回登陆点 — 调用方法

```csharp
// CommonlyPanel.OnClickBtns(4) 确认后：
if (BattleDataHolder.IsInBattle)
    battleProcesser.ForceExitBattle();   // 若在战斗中先强退
```

| 项目 | 值 |
|------|-----|
| 协议 | `LSSPROTO_LOGINGATE_FUNC = 60` |
| 消息体 | `Proto_CS_Logingate`（空） |

**限制：** 战斗中提示「战斗中暂时无法回记录点哟」；队员非队长不可操作（遇敌相关按钮同理）。

### 8.3 原地遇敌 — 开启 / 取消

统一走 **`RoleManager.SendAutoBattle`**：

| 项目 | 值 |
|------|-----|
| 协议 | `LSSPROTO_AUTOBATTLE_FUNC = 3015` |
| 消息体 | `Proto_CS_AutoBattle`（`Type` + `KUid`） |

**CommonlyPanel 逻辑（index 2）：**

```csharp
if (encounterStatus == 0)
{
    SendAutoBattle("开始挂机", uid);
    TeamManager.isAutoBttle = true;
}
else
{
    SendAutoBattle("停止挂机", uid);   // 取消遇敌
}
```

**全部 Type 字符串：**

| Type | 作用 |
|------|------|
| `"开始挂机"` | 开启原地遇敌 |
| `"停止挂机"` | **取消遇敌 / 完全退出挂机** |
| `"暂停挂机"` | 暂停遇敌（仍算挂机态） |
| `"继续挂机"` | 从暂停恢复 |
| `"结束计时"` | 结束挂机计时（移动地图等触发） |
| `"奖励查看"` | 查看挂机奖励 |

### 8.4 当前是否处于遇敌中

**主字段：** `PlayerData.encounterStatus`（按角色）

```csharp
// 主控角色
int st = PlayerDataHolder.playerData.encounterStatus;

// 指定多开角色
int st = PlayerDataHolder.GetPlayerFromUid(uid).encounterStatus;
```

**服务端同步：** `Proto_SC_UpdatePlayer` → `PROTO_PLAYER_TYPE_ENCOUNTERSTATUS (64)` → 写入 `encounterStatus`。

| 值 | 含义 | UI 表现 |
|----|------|---------|
| **0** | 未挂机 / 未遇敌 | 按钮「原地遇敌」 |
| **1** | **遇敌进行中**（正常挂机） | 按钮「取消遇敌」；可发「暂停挂机」 |
| **2** | **遇敌已暂停** | 仍 `!=0`；可发「继续挂机」恢复 |

**判断公式：**

```csharp
bool inEncounter = (encounterStatus != 0);           // 广义「挂机/遇敌态」
bool encounterActive = (encounterStatus == 1);       // 正在遇敌（非暂停）
bool encounterPaused = (encounterStatus == 2);       // 暂停中
```

**辅助标志（客户端）：**

| 字段 | 说明 |
|------|------|
| `TeamManager.isAutoBttle` | 点击开始后置 true，停止后 false |
| `RoleManager.OnHookStatus` | 挂机计时 UI 用 |
| `OnHookWordPanel` | `encounterStatus!=0` 时自动打开 |

**副作用：** `encounterStatus != 0` 时移动、任务寻路、部分操作会提示「自动挂机中无法行动！」；地图上点击移动且 `encounterStatus==1` 会自动发 `"停止挂机"`。

### 8.5 如何解除原地遇敌

**完全退出（推荐，等同点「取消遇敌」）：**

```csharp
Manager<RoleManager>.Instance.SendAutoBattle("停止挂机", uid);
Manager<TeamManager>.Instance.isAutoBttle = false;
```

服务端回包后 `encounterStatus` 变为 **0**，`OnHookWordPanel` 关闭。

**仅暂停（仍占挂机态，`encounterStatus` 变为 2）：**

```csharp
SendAutoBattle("暂停挂机", uid);   // 仅当 encounterStatus == 1
```

**恢复遇敌：**

```csharp
SendAutoBattle("继续挂机", uid);   // 当 encounterStatus == 2
```

### 8.6 助手 Bridge 示例

```csharp
// 开启原地遇敌

// 取消原地遇敌
RoleManager.SendAutoBattle("停止挂机", uid);

// 回登陆点

// 读状态

```

### 9.2 超值月卡 — 每角色每日领取状态

#### A. 快速判断：月卡是否在有效期内

| 字段 | 来源 | 含义 |
|------|------|------|
| `PlayerData.MonthCardTime` | `Proto_OnePlayerData.PrivilegeTime` | 月卡到期 Unix 时间戳 |
| `PlayerData.MonthCardOpen` | `GetServerTime() < MonthCardTime` | **true = 月卡有效** |

```csharp
PlayerData p = PlayerDataHolder.GetPlayerFromUid(uid);
bool cardActive = p.MonthCardOpen;
int expireAt  = p.MonthCardTime;
```

> 仅表示「有没有月卡」，**不能**单独判断「今日奖励是否已领」。

#### B. 精确判断：今日每日奖励是否已领

需请求活动详情，读 `Proto_SC_Activity`（`Type == "特权月卡"`）：

| 字段 | 类型 | 含义 |
|------|------|------|
| `TqkTime` | `int` | 活动侧月卡到期时间（与 `MonthCardTime` 同源，UI 倒计时用） |
| `TqkbDay` | `bool` | **true = 今日每日奖励已领取**；**false = 未领（可领）** |
| `ActivityId` | `int` | 领取时必传 |
| `KUid` | `string` | 角色 uid |

**UI 逻辑（`MonthCardChildPanel`）：**

```csharp
// 月卡有效
bool active = m_ActivityInfo.TqkTime - TimeManager.GetServerTime() > 0;

// 显示「领取」按钮
bool showClaimBtn = !m_ActivityInfo.TqkbDay;

// 已领则隐藏按钮；服务端回包 Type="领取每日奖励" 时再次刷新

```

**查询流程（每个角色）：**

```
1. SendActivity("活动列表", uid)
   ← Type="活动列表", Titles[] = { Id, Name, Seqno }

2. 在 Titles 中找到「特权月卡 / 超值月卡」对应 Id
   （Id 由服务端下发，非硬编码；需按 Name 匹配或试开）

3. SendActivity("活动信息", uid, titleId)
   ← Type="特权月卡", TqkTime, TqkbDay, ActivityId, KUid
```

**可领条件：**

```csharp
int now = TimeManager.GetServerTime();
bool canClaimDaily = (activity.TqkTime > now) && !activity.TqkbDay;
```

#### C. 如何领取每日奖励

```csharp
Manager<ActivityManager>.Instance.SendActivity(
    "领取每日奖励",
    uid,
    id: 0,
    activityId: activity.ActivityId
);
```

**服务端回包：** `Proto_SC_Activity`，`Type == "领取每日奖励"`，`TqkbDay == true` 表示成功已领。

**配置表：** 每日奖励内容来自 `TbAdvVipCardConfig`（`active_tbadvvipcardconfig`）的 `DayGet` 列表；充值开通走 `RechargeConfig` Id=**27**（`RechargeTipType.yueka`）。

#### D. 月卡权益开关（非每日领取）

`MonthCardChildPanel` 关闭时会同步四项权益到服务端：

```csharp
// totalStatus[0..3] = 自动销毁 / 远程卖魔石 / 自动维修 / 自动治疗
```

服务端回 `Proto_SC_MonthCard`，`Type == "更新权益状态"` → 打开/刷新月卡子面板。

## 10. 背包物品：按名称查找 / 使用 / 丢弃

> 背包数据按 **角色 uid** 存于 `PlayerDataHolder`；协议操作一律传 **背包格子 index（Haveitemindex）**，**不传物品名称**。

### 10.1 背包数据结构

```csharp
List<ItemData> bag = PlayerDataHolder.GetItemDatasFromUid(uid);
```

| 索引范围 | 含义 |
|----------|------|
| **0 ~ 7** | 已穿戴装备栏 |
| **8 ~ itemGridNum - 1** | 背包道具格（`PlayerData.itemGridNum` ← `Proto_OnePlayerData.ItemGridNum`） |

每个 `ItemData`：

| 字段 | 说明 |
|------|------|
| `useFlag` | **1** = 格子里有物品 |
| `data` | `Proto_ItemData`，含 `Id`、`Index`（格子号）、`Name`、`Pile`（堆叠数）、`Type` 等 |
| `data.Index` | 与列表下标一致，即 **Haveitemindex** |

**名称来源：** 服务端同步后 `Proto_ItemData.Name` 会被设为配表 **`ItemConfig.Secretname`**（UI 显示名）。聊天附件校验也是 `value == itemData.data.Name`（见 `ChatManager.formatMessage`）。

配表：

```csharp
ItemConfig cfg = Manager<ConfigManager>.Instance.GetTbItemConfig().GetOrDefault(itemId);
string displayName = cfg.Secretname;  // UI 名称
string internalName = cfg.Name;         // 配表内部名（较少用于背包匹配）
```

### 10.2 按名称判断背包中是否有物品

客户端 **没有** `GetItemByName` 之类封装，需自行遍历或使用 Id。

#### A. 按显示名称（Secretname）遍历

```csharp
ItemData FindItemByName(string uid, string itemName)
{
    var items = PlayerDataHolder.GetItemDatasFromUid(uid);
    int gridNum = PlayerDataHolder.GetPlayerFromUid(uid).itemGridNum;
    for (int i = 8; i < gridNum; i++)
    {
        ItemData slot = items[i];
        if (slot != null && slot.useFlag == 1 && slot.data.Name == itemName)
            return slot;
    }
    return null;
}
```

> 同名多格会返回第一个匹配格；需指定某一格时继续比较 `Index` 或 `Pile`。

#### B. 按物品 Id（推荐，更稳定）

```csharp
// 已有封装：统计某 Id 总数量（含堆叠）
int count = Manager<ItemManager>.Instance.GetItemNumByUid(uid, itemId);

// 找第一个该 Id 的格子
List<ItemData> FindItemsById(string uid, int itemId)
{
    var result = new List<ItemData>();
    var items = PlayerDataHolder.GetItemDatasFromUid(uid);
    int gridNum = PlayerDataHolder.GetPlayerFromUid(uid).itemGridNum;
    for (int i = 8; i < gridNum; i++)
    {
        ItemData slot = items[i];
        if (slot?.useFlag == 1 && slot.data.Id == itemId)
            result.Add(slot);
    }
    return result;
}
```

#### C. 名称 → Id（若只有中文名）

```csharp
int? FindItemIdBySecretName(string secretName)
{
    foreach (var cfg in Manager<ConfigManager>.Instance.GetTbItemConfig().DataList)
    {
        if (cfg.Secretname == secretName)
            return cfg.Id;
    }
    return null;
}
```

#### D. 按类型筛选

```csharp
List<ItemData> byType = Manager<ItemManager>.Instance.GetItemByType(uid, itemType);
// 仅扫描背包格 8+，再按 data.Type 过滤
```

### 10.3 使用物品

#### 协议

| 项目 | 值 |
|------|-----|
| 协议号 | `LSSPROTO_USE_ITEM_FUNC = **16**` |
| 消息体 | `Proto_CS_UseItem` |

**API：**

```csharp
Manager<ItemManager>.Instance.SendUseItem(
    x, y,              // 角色当前坐标 PlayerDataHolder.location
    haveitemindex,     // 背包格子 index（不是物品 Id）
    uid,
    toindex = 0,       // 目标 index（如对宠物使用时 petindex+1）
    selectIndex = -1,  // 自选礼包等子选项
    useNum = 1         // 使用数量（堆叠道具）
);
```

**背包 UI 等价调用（`ChildBackPackPanel`）：**

```csharp
// 普通使用
SendUseItem(location.x, location.y, index, uid);

// 批量使用（先弹数量框）
SendUseItem(x, y, useIndex, uid, toindex: 0, selectIndex: choseId, useNum: num);
```

**使用前客户端检查（双击背包）：**

- 格子在 `itemGridNum` 内且 `useFlag == 1`
- 通常需已鉴定：`data.Flg.HasFlag(PROTO_ITEM_FLAG_APPRAISAL)`
- 按 `data.Type` 分流：自选礼包(64)、灵装(66)、宝石袋(61) 等走专用逻辑，否则 `Type < 23` 或 `TypeUseFlag` 才允许使用

**需要对目标使用（选宠物/玩家）：**

```csharp
// LSSPROTO_BACKPACK_FUNC = 2001
Manager<ItemManager>.Instance.SendUseItemToTargetMessage(
    "使用道具到指定对象",
    haveitemindex,
    targetIndex,
    uid
);
// 服务端先回 Proto_SC_Backpack Type="选择对象数据" 打开 SelectPanel
```

### 10.4 丢弃物品

#### 背包内丢弃（游戏「丢弃」按钮）

| 项目 | 值 |
|------|-----|
| 协议号 | `LSSPROTO_BACKPACK_FUNC = **2001**` |
| Type 字符串 | **`"丢弃道具"`** |
| 参数 | **`Haveitemindex` = 格子 index**，**`Num` = 丢弃数量**，**`KUid` = uid** |

```csharp
// ChildBackPackPanel.DropItem — 不传名称
Manager<ItemManager>.Instance.SendBackPackMessage("丢弃道具", itemIndex, num, uid);
```

**结论：丢弃发的是背包格子 index + 数量，不发物品名称。**

#### 扔到地上（另一套协议，hotfix 内几乎未调用）

```csharp
// LSSPROTO_DROP_ITEM_FUNC = 18 — SendItemDrop 在反编译代码中无调用点
Manager<ItemManager>.Instance.SendItemDrop(x, y, itemindex, uid);
```

日常背包「丢弃」走 **2001 背包协议**，不是 **18 丢地协议**。

| `"拆分道具"` | 拆分堆叠 |
| `"整理背包"` | 整理 |
| `"一键鉴定"` | 批量鉴定 |
| `"增加道具栏"` | 扩格 |
